Create a responsive, mobile-first Web Application for tracking office attendance and planning upcoming office days. The application must be highly optimized for both mobile screens and desktop web browsers.

### Tech Stack Preferences
- Frontend: React with Tailwind CSS for styling, Lucide React for icons.
- UI Components: Clean, modern, minimal design (similar to shadcn/ui aesthetics).
- Backend/Storage: Simple Express.js backend with a local database (like SQLite or lowdb) or browser localStorage if a standalone backend isn't strictly necessary, but prioritize a lightweight persistent backend so data isn't lost on cache clear.

### Core Architecture & State Management
Every calendar date can have one of the following mutually exclusive states:
1. "Present" (Attended office)
2. "Company Leave" (Public holiday / office closed)
3. "Personal Leave" (Sick leave / Casual leave / Vacation)
4. "Planned Office Day" (A future-facing state: "will be in office")
5. "Remote/Default" (Worked from home / unrecorded)

### View 1: Homepage (Quarterly Dashboard)
This is the default landing view. It must feature a clean dashboard showing the current calendar year broken down into four distinct quarterly cards:
- Q1: Jan - Mar
- Q2: Apr - Jun
- Q3: Jul - Sep
- Q4: Oct - Dec

Each Quarterly Card must dynamically calculate and display:
- Total number of weeks in that specific quarter.
- "Days in Office": Total count of "Present" states registered within that quarter till date.
- A subtle visual progress bar or metric showing office attendance density.
- Quick-action button: A prominent toggle or floating button on the homepage to "Check-in to Office Today" (instantly marking today's date as "Present").

### View 2: Calendar View & Attendance Logger
A clean, interactive monthly calendar grid (switching between months using next/prev arrows).
- Visual Feedback: Each day cell in the calendar must be color-coded or icon-marked based on its state (e.g., subtle green dot/background for Present, soft red for Personal Leave, gray for Company Leave, and a distinct dashed/blue outline border or badge for "Planned Office Day").
- Logging Functionality: Clicking on any date cell must open a clean modal or bottom sheet (on mobile) that allows the user to select or change the state of that day using an intuitive radio/button group:
  - [ ] Attended Office ("Present")
  - [ ] Company Leave
  - [ ] Personal Leave
  - [ ] Plan to be in Office ("Planned Day")
  - [ ] Reset to Default / Remote
- Include a quick note field for each day if the user wants to add context (e.g., "Deep work day" or "Sick leave").

### Navigation & UX Requirements
- Implement a bottom navigation bar for mobile screens and a sleek top header/sidebar navigation for desktop screens to seamlessly switch between "Dashboard" and "Calendar".
- Ensure all interactions are touch-friendly with distinct active/hover states.
- Pre-populate the app with mock data for the current running quarter so the dashboard metrics and calendar views look alive immediately upon first build.
